# 現在のサイトのフロント仕様書

## 1. サイト概要
- 2026 Adultopia 大人國のクリエイター投票／募集向け単一ページアプリケーション（SPA）。
- GitHub Pages（`https://adultopia.github.io/creator-vote`）にデプロイされ、`HashRouter` でルーティング。
- 現在は投票 UI のプロトタイプ段階で、モックデータと簡易アラートによる確認導線のみ実装。

## 2. 技術スタック
- **フレームワーク**: React 19 + CRA（react-scripts 5）。
- **スタイリング**: styled-components（テーマ・グローバルスタイル）、Tailwind CSS CDN（ユーティリティクラス適用）。
- **国際化**: i18next + react-i18next + LanguageDetector（日本語／繁体字中国語／韓国語）。
- **ルーティング**: react-router-dom 7（HashRouter + Routes）。
- **メタ管理**: react-helmet（フォント・favicon 等）。

## 3. ルーティングとエントリーフロー
- エントリーポイントは [src/index.tsx](src/index.tsx)。`React.StrictMode` 配下で `<App />` をマウント。
- [src/App.tsx](src/App.tsx) で `ThemeProvider`・`GlobalStyle` を適用し、`HashRouter` 内に `/` ルートを定義。現状ページは `HomePage` のみ。
- `Helmet` で Google Fonts（Inter / Noto Sans JP）を事前接続し読み込む。

## 4. デザインとビジュアル仕様
### 4.1 タイポグラフィ
- ベースフォント: `'Noto Sans JP', 'Inter', sans-serif`（`GlobalStyle` 経由）。
- フォントウェイト: 400 / 500 / 700（Google Fonts で読み込み）。

### 4.2 カラーパレット
| 役割 | 値 | 用途 |
| --- | --- | --- |
| Primary | `#FF75B2` | CTA、カード選択枠、強調用（Tailwind では `text-pink-500` なども併用）。|
| Primary Dark | `#E55D9A` | ホバーや濃色系アクセント想定。|
| White | `#FFFFFF` | 背景・カード・オーバーレイ。|
| Black | `#000000` | ベーステキスト。|
| Gray | `#F5F5F5` | グローバル背景、セクション。|
| Light Gray | `#EEEEEE` | 枠線の補助色。|

### 4.3 背景・レイアウト
- ルート背景: `public/adultopia/hero.jpg` を `cover / center / fixed` で敷き、白の半透明レイヤー（`bg-white/50`）＋ `backdrop-blur` を重ねて可読性を確保。
- 全体は `min-h-screen` の縦フレックス構造。ヘッダー固定、下部にアクションバーを条件表示。

## 5. ページ構成と主要コンポーネント
### 5.1 HomePage（[src/pages/HomePage.tsx](src/pages/HomePage.tsx)）
- **データ**: `USER_STATUS`（ログイン状態＋残票数）、`CREATORS`（30名のモック、picsum 画像、累計票数）。
- **状態**: `selectedIds`（投票候補 ID 配列）。最大選択数は残票数 (`remainingVotes`)。
- **ヘッダー**: 固定ヘッダーにロゴ画像（`public/adultopia/logoYoko.png`）と残票カウンター。背景は半透明白＋ブラー。
- **カードグリッド**: `grid-cols-2`（モバイル）→`md:grid-cols-4`。カードは `button`要素、選択時は `border-[3px]` + `ring` + `#FF69B4`。リミット到達時、未選択カードは `opacity-50` で無効化。
- **CTA バー**: `selectedIds.length > 0` で下部に固定表示。`max-w-md` の丸ボタンに投票人数を表示し、クリックで `alert` を発火。

### 5.2 Footer（[src/components/Footer.tsx](src/components/Footer.tsx)）
- styled-components ベースの 2 カラム構造（ロゴ／インフォメーション）。768px 以下で縦積み。
- 文言は `useTranslation()` 経由で取得。ロゴは `logoYoko.png` を利用。
- **現状**: `HomePage` から呼び出されておらず、UI には未表示。

### 5.3 DescriptionTitleWithImage（[src/components/DescriptionTitleWithImage.tsx](src/components/DescriptionTitleWithImage.tsx)）
- テキスト横に 32px アイコンを配置する汎用説明ブロック。
- 未使用コンポーネント。

### 5.4 GlobalStyle / Theme（[src/styles/GlobalStyle.ts](src/styles/GlobalStyle.ts), [src/styles/theme.ts](src/styles/theme.ts)）
- 全要素のリセットとフォント、ボックスサイズを統一。
- テーマには `colors` / `fonts` / `breakpoints` を定義。`ThemeProvider` 経由で利用可能。

## 6. 状態管理・ビジネスロジック
- `toggleSelect(id)`:
  - 既に選択済みなら解除。
  - 未選択かつ `selectedIds.length < maxVotes` の場合のみ追加。
  - 上限を超える操作は無視（UI ではカード自体を `pointer-events-none` で無効化）。
- `remaining = maxVotes - selectedIds.length` をヘッダーに表示、0 でピンク強調。
- `handleConfirm()` は選択 ID を `alert` 表示するのみで、API 連携は未実装。

## 7. 多言語対応
- [src/i18n.ts](src/i18n.ts) で `zh-TW`／`ja`／`ko` リソースを設定。検出順は `querystring(lng)` → `localStorage` → ブラウザ設定。未検出時は `zh-TW`。
- 文字列は [src/locales/*.json](src/locales/ja.json) に定義。現在はフォーム系文言が中心。
- **注意**: `i18n` 初期化ファイルがどこからも import されておらず、実行時に翻訳リソースが読み込まれない。結果として `useTranslation()` はキーをそのまま返す。

## 8. アセットとメディア
- `public/adultopia/hero.jpg`: 背景画像（固定、全面）。
- `public/adultopia/logoYoko.png`: ヘッダー／フッターロゴ。
- `https://picsum.photos/seed/creator-{id}/400/400`: カードのプレースホルダー画像。
- `public/OGP.PNG`, `icon_nobg.png`, `manifest.json` 等を OGP / PWA 用に配置。

## 9. レスポンシブ仕様
- Tailwind ユーティリティで制御。主なブレークポイント: `md`（768px）。
  - グリッド: モバイル2列 → mdで4列。
  - ヘッダー／ CTA バーは幅 `max-w-6xl` で中央寄せ。
- styled-components 側でもフッター等で 768px / 480px を基準にレイアウト切替。

## 10. ビルド・デプロイ
- `npm run build` で CRA ビルド。`homepage` 設定により GitHub Pages 用パスを解決。
- `npm run deploy` は `gh-pages -d build` を呼び出し、`predeploy` ビルド後に公開。
- 開発は `npm start`（`start:lan` で LAN 共有も可能）。
